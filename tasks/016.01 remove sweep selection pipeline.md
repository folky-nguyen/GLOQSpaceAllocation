# 016.01 Remove Sweep Selection Pipeline

## Goal

In `apps/web`, remove the full sweep-selection pipeline so the editor shell keeps one default click-based selection flow instead of carrying a second drag-select mode.

## Required Outcome

- remove the `sweep` select mode from the UI store
- remove the `Sweep Select` option from the ribbon `Select` menu
- remove the floor-plan pointer pipeline that starts, tracks, and commits drag selection rectangles
- remove the sweep overlay box and sweep-only cursor styling
- keep default click selection working for single and multi-space selection
- keep `Select All Visible` and `Clear Selection` working inside the surviving `Select` menu
- update shell copy so it no longer references a selection mode that no longer exists

## Constraints

- keep the change local to the web shell
- do not change `ProjectDoc`, persistence, renderer contracts, or Supabase schema
- prefer deleting dead selection-mode state over replacing it with a new abstraction
- keep one shared selection model for 2D and 3D emphasis

## Current Repo State

### `apps/web/src/ui-store.ts`

Today the UI store still owns a second selection-mode seam:

- `SelectMode = "pick-many" | "sweep"`
- `selectMode`
- `setSelectMode(...)`

That state only exists to drive alternate shell behavior and should disappear if sweep selection is removed.

### `apps/web/src/editor-shell.tsx`

Today this file still carries the full sweep pipeline:

- `selectModeItems`
- `getSelectModeLabel(...)`
- `getSelectModeHint(...)`
- `SweepMode`
- `SweepSelectionDraft`
- `getSweepMode(...)`
- `getSweepSelectionBounds(...)`
- `getSweptSpaceIds(...)`
- `mergeSpaceSelection(...)`
- `showSelectMenu` mode buttons for `Pick Many` and `Sweep Select`
- plan-canvas pointer handlers and overlay rendering for the drag box
- properties/status copy that still says `Select mode`

### `apps/web/src/styles.css`

Today this file still styles sweep-only plan behavior:

- `.plan-canvas.is-sweep-mode`
- `.plan-sweep-box`
- `.plan-sweep-box.is-add`
- `.plan-sweep-box.is-remove`

## Scope

In scope:

- deleting sweep-only store state
- deleting sweep-only menu items and copy
- deleting sweep-only canvas handlers and overlay rendering
- deleting sweep-only CSS
- simplifying status/properties copy to match the surviving click-selection flow

Out of scope:

- changing the underlying `Selection` union
- redesigning the ribbon or properties panel
- changing site-edge selection behavior
- changing 3D selection emphasis behavior

## Implementation Decisions

### 1. Keep click selection as the single surviving flow

Lean rule:

- keep `toggleSpaceSelection(...)`
- remove the alternate drag-select mode instead of introducing a replacement state

Reason:

- click-based multi-selection already exists and is enough for the current MVP
- removing the second flow simplifies the shell more than trying to keep both paths partially alive

### 2. Delete mode state from the store instead of freezing it

Lean rule:

- remove `SelectMode`, `selectMode`, and `setSelectMode(...)` from `ui-store.ts`

Reason:

- if sweep is gone, a one-value mode store is dead state
- deleting the state is clearer than keeping a permanent `"pick-many"` constant in the store

### 3. Keep the `Select` menu as an actions menu only

Lean rule:

- remove the mode-switch section from the `Select` menu
- keep action items such as `Select All Visible` and `Clear Selection`

Reason:

- the menu still has value as an action surface
- it should stop pretending there are multiple selection pipelines

### 4. Replace mode-specific copy with fixed guidance

Lean rule:

- remove `Select mode` rows from properties and status
- keep one short static hint for plan selection behavior

Reason:

- once sweep is removed, mode labels become stale UI noise
- fixed guidance is smaller and easier to maintain

## Acceptance

- no `Sweep Select` option remains in the shell
- no `selectMode` store state or setter remains
- plan view no longer starts or renders a sweep selection rectangle
- clicking spaces still adds and removes them from the current selection
- `Select All Visible` and `Clear Selection` still work
- status/properties copy no longer references `Select mode`
- build passes after the cleanup
