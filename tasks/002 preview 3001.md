Thiết lập chế độ preview/dev chạy ở cổng 3001 cho repo hiện tại.

Yêu cầu:
1. Tìm đúng entry dev server hiện tại (Next.js / Vite / app web package).
2. Đổi hoặc bổ sung cấu hình để preview chạy ổn định ở port 3001.
3. Không hardcode rải rác nhiều chỗ. Ưu tiên 1 nguồn cấu hình rõ ràng.
4. Nếu cần thêm file config, script, env, hoặc tài liệu cài đặt:
   - tạo và gom chúng vào một thư mục riêng để dễ quản lý
   - ưu tiên tên thư mục như `setup/`
   - giải thích ngắn trong README của thư mục đó từng file dùng để làm gì
5. Nếu có file `.env`, `.cmd`, `.ps1`, `.sh`, hoặc script hỗ trợ local preview thì đặt chúng trong thư mục riêng đó, trừ khi framework bắt buộc file phải ở root.
6. Nếu framework bắt buộc một số file phải ở root, thì:
   - giữ file root ở mức tối thiểu
   - phần còn lại vẫn đặt trong thư mục setup riêng
   - thêm comment chỉ rõ vì sao file đó phải ở root
7. Cập nhật `package.json` scripts để tôi có thể chạy rõ ràng như:
   - `pnpm dev:3001`
   - hoặc `pnpm preview:3001`
8. Không phá vỡ luồng dev hiện tại ở cổng cũ nếu repo đang dùng mặc định.
   - Giữ backward compatibility nếu hợp lý.
9. Sau khi sửa xong, xuất ra:
   - danh sách file đã tạo/sửa
   - lệnh để chạy preview ở cổng 3001
   - giải thích ngắn kiến trúc cấu hình mới

Tiêu chuẩn thực hiện:
- tối giản
- dễ tìm
- dễ bảo trì
- không duplicate config
- không thêm dependency nếu chưa cần

Nếu repo là monorepo/workspace:
- chỉ chỉnh package web/app liên quan
- đảm bảo script root gọi đúng package đó
- nếu cần, thêm script root để gọi package con chạy ở port 3001

Hãy tự đọc repo trước, xác định đúng chỗ cần sửa rồi mới patch.

## Context cập nhật sau khi triển khai

- Repo là `pnpm` workspace.
- Entry dev server hiện tại là package `apps/web` dùng `Vite`.
- Root script hiện hành gọi web package qua `corepack pnpm --filter web ...`.
- Để giữ backward compatibility, luồng mặc định vẫn là `vite` trên cổng cũ và không đổi `apps/web/vite.config.ts`.
- Luồng mới cho cổng `3001` được nối theo một tầng rõ ràng:
  - root: `pnpm dev:3001`
  - web: `vite --port 3001 --strictPort`
- Chỉ cần sửa `package.json` ở root và `apps/web/package.json`; chưa cần thêm `setup/` vì chưa phát sinh file cấu hình phụ.

## Kinh nghiệm / ghi chú vận hành

- Với bài toán này, cấu hình port nên đặt ở script của package web thay vì nhân đôi thêm vào nhiều file.
- `--strictPort` là cần thiết để fail fast khi cổng bận, tránh Vite tự nhảy sang cổng khác và làm sai kỳ vọng local preview.
- Khi cổng `3001` bị chiếm, quy trình chẩn đoán đủ gọn là:
  - `Get-NetTCPConnection -LocalPort 3001 -State Listen`
  - `Get-CimInstance Win32_Process -Filter "ProcessId = <pid>"`
  - `Stop-Process -Id <pid> -Force`
- Trường hợp thực tế trong máy local: cổng `3001` bị một tiến trình `node` từ repo khác (`GitHub\\Map`) chiếm do đang chạy `vite preview`.
- Sau khi giải phóng cổng, `pnpm dev:3001` của repo này chạy đúng trên `http://127.0.0.1:3001` và trả HTTP `200`.
